# 文脈つき単語訳の設計（2026-07-06 確定・/decide 討論の結論）

## 問題（実コードで確定済み）
1. クリック語の日本語訳が**単語1語だけ**で DeepL/Azure に送られ文脈ゼロ → 多義語が場面と無関係な訳になる（実例: CONFIRMATION→「堅信式」。文脈では「確認」）。
   - extension/content.js `translateToJa(word)` — sentence は手元にあるのに API に渡していない
2. **キャッシュ汚染構造**: translation_cache のキーが word+lang のみ → 一度の誤訳が全ユーザー・全作品に永続配布。
3. **ja フィールド欠落**: 保存 entry に日本語訳が無く（definition=英英のみ）、単語帳は fetchJa(word) で**別の1語訳を取り直す** → ポップアップの訳と単語帳の訳が別物になり得る。復習で誤訳を刷り込む最悪経路。
   - content.js entry(≈L810)・lib/supabase.js my_words マッピング(L214-225)・WordbookScreen.js(L68) で裏取り済み
4. 英英定義も meanings[0].definitions[0] の第一語義のみ。

## 討論の経過（engineer × user-voice・R1独立→R2敵対）
- engineer が R2 で2点撤回: 「未ログインは速報のみ」→ 無料層にも文脈訳を開放（キャッシュとIP上限で守る）／キャッシュキー word+tmdbId+S/E → **sense_hash 主軸**（同一エピソード内の同語多義=法廷ドラマの brief 等が混ざるため）。
- user-voice が R2 で譲歩: 未ログイン絞りは導線実態から許容（条件=速報訳は出した上で補助線文言）／「初回だけ待つ」を支持強化（500-900msなら）。
- 残存した強い要求: **過渡期（ポップアップ誤訳・単語帳正確の不整合）を数週間で終わらせること**。

## 確定設計

### 1. 生成方式
- **Claude Haiku に「字幕文＋語」を渡し、その場面での語義を返す**（15字以内・名詞句・説明なし）。
- DeepL文訳＋語抽出案は不採用（語順・訳し上げで対象語の対応が取れない）。
- 実装は新 route を切らず **`/api/claude` に `mode:'wordsense'` を追加**し、backlog 赤タスク「サーバ側強制」（用途別 max_tokens 上限・プロンプト検証）と**統合実装**する。語義モードは max_tokens 64 上限。
- トークン ≈135/回 → **1回 ≈ ¥0.05〜0.1**（「AI解説=¥1/文」とは別物。語義だけなので桁が下）。

### 2. 表示（クリックポップアップ）
- キャッシュ命中 → **即時表示**（大半のケース。共有キャッシュが効く）。
- 未命中 → スケルトン（検索中）→ **Haiku が ~800ms 以内なら確定訳のみを表示**（「間違い→訂正」の二度読みを構造的に排除）。
- Haiku が遅い場合 → DeepL 速報訳を**薄色の暫定表示** → Haiku 到着で同位置確定。**1.5s タイムアウトで速報訳を無言で確定**（エラー文言は出さない。ユーザーは理由より意味が知りたい）。
- 表示形式は **「この場面では: 確認」の1行**。長文解説は書かない（それは有料AI解説の領分）。キャッシュ由来（他ユーザーのクリックで生成済み）でも同じ表現でよい。
- ホバーのチラ見は**従来の1語速報のまま**（連続発火のCOGS・重要度最低）。

### 3. キャッシュ
- 新テーブル **`translation_ctx_cache`**。主キー = `word + target_lang + tmdb_id + sense_hash`（sense_hash = 正規化した字幕文の sha1）。season/episode は付帯メタ。
- sense_hash 主キーにより同語多義の誤配布を構造的に防止。ヒットは「同じ字幕行」に限られるが、厳選カタログ＝同じ場面を多人数が観る前提で共有キャッシュが効く（shared-cache-design と同じ原理）。
- 既存 `translation_cache` は**「文脈なし速報」用と再定義**（削除不要・ホバーとフォールバックで継続使用）。誤訳が問題化した語だけ個別上書き。

### 4. 無料/有料と COGS 保護
- **文脈訳は無料コア**（単語の意味＝最重要要素・無料版の精度が信頼の最低ライン）。未ログインにも開放。
- 守り: ①キャッシュ命中は無条件・無償配布 ②**新規 Haiku 生成のみ IP 日次上限（50回/日）** ③既存 IP レート制限（30/分）。
- COGS 試算（命中率込み）: 限定公開50人 ≈ **月¥1,350**／1000人 ≈ **月¥13,500**（命中率85%+想定）。許容範囲。
- 有料との線引き維持: 「場面内の語義1行」=無料 ／ 「文全体のAI解説（文法・ニュアンス）」=有料（#17確定のまま）。この線引きは user-voice 評価で「自然」。

### 5. ja の保存（訳の固定）
- **保存時にポップアップで見せた文脈訳を entry に固定保存**（表示時に都度引く案は却下=レイテンシとCOGSが表示回数比例）。
- スキーマ: my_words に **`ja` 列新設**（definition=英英と意味が別なので統合しない）。⚠GRANT 明示を忘れない（infra-gotchas）。
- 変更点: content.js entry に `ja`（Promise.all で取得済みの値）／background.js 同期ペイロード／supabase.js push/pull マッピング／WordbookScreen は `w.ja ?? fetchJa(word)` フォールバック。
- 効果: **ポップアップで見た訳＝単語帳・復習カードの訳**が一致（user-voice 最重大要件）。

### 6. 出荷順（過渡期を最短化）
1. **サーバ＋アプリ先行（拡張不要・即デプロイ可）**: wordsense route（サーバ側強制と統合）・translation_ctx_cache・my_words ja列・単語帳の文脈訳化（保存済み語は sentence を持つので遡って文脈訳を引ける）。
2. **拡張 v1.2.2**（v1.2.1 審査通過後すぐ申請）: sentence/ctx を API へ引き回し・ja 固定保存・ポップアップの新表示・（同梱: 監査修正済み分＋シネマロゴアイコン統一）。
3. 過渡期（ポップアップ旧動作・単語帳のみ正確）は **v1.2.2 の審査期間だけ**に留める（user-voice 条件「数週間以内」を満たす）。
