# 厳選カタログ戦略の設計（2026-07-06 /decide討論の結論・オーナー最終承認待ち）

予習の単語リスト生成を「運営整備のカタログ作品＋リクエスト駆動の週次追加」に切り替える戦略。
user-voice / legal / marketing の3部署 R1独立→R2的絞り検証で収束。

## 動機（検証済みの数字）
- 生成COGSが「ユーザー行動次第＝予測不能」から「運営の意思決定」に変わる。
- 1,000人・転換4%の月次損益: 現行 +¥8,000（最悪 −¥21,000）→ カタログ制 **≈+¥14,000（最悪 ≈+¥11,000）**。
  **最大の効果は利益額より赤字シナリオの消滅**（分散ゼロ化）。
- 初期ライブラリ整備 ≈¥6,000一時＋月¥1,000〜2,000（週次追加分）。

## 確定設計

### 1. 制限の範囲（重要）
- **どの作品でも動く**: 字幕クリック保存・文脈つき訳（wordsense）・復習・単語帳＝コアループ無傷。
- **カタログ作品のみ**: 予習の単語リスト生成（＝コストの主役・生成¥7/話）。
- カタログは無料/有料共通（コンテンツアクセス課金のグレーに触れない＝既決の線引き維持）。

### 2. ソフト版（ハード版は不採用）
- カタログ外の作品では「リクエスト受付中」ボタン＋クリック保存の継続案内。
- ハード版（リクエスト経路なし）は法務が非推奨（「特定作品を狙った利用」の心証悪化）＋
  user-voice が離脱リスクと判定。
- **言葉の掟（罪悪感UI原則の言語適用）**: 「この作品は非対応です」は禁止。
  「リクエスト受付中」「対応予定一覧」で巻き込む。突き放しの表現を全排除。

### 3. リクエスト運用（user-voiceの絶対条件）
- リクエスト→**翌週までに対応 or 理由つき反応**を運営義務とする（無反応は信頼崩壊＝最大の離脱要因）。
- 「対応予定一覧」を画面に可視化（見えることで信頼が生まれる）。
- リクエストランキングの表示（「〇〇 現在△票」）＝需要データ＋共感の接着剤（小規模Friend Streak効果）。
- ⚠この週次SLAはオーナーの作業コミット＝**採用前にオーナーの体力確認が必要**（エスカレーション事項）。

### 3b. 有料特典「優先リクエスト」（2026-07-06 追加決定）
- ❌**「有料＝リクエスト不要で任意作品を即生成」は不採用**：既決の法務ライン
  （コンテンツアクセス課金のグレー回避・作品制限を課金主軸にしない）に正面から触れ、
  「作品カバレッジの販売」の外形になる。legal R2 の量統制（週30話・分散）もセルフ生成では守れない。
- ⭕**採用＝「優先リクエスト」**：有料はリクエストがキューの先頭（例: 翌日対応の目安）。
  売るのは生成権でなく**順番（サービス優先度）**。生成は運営主体のまま量統制の枠内。
  **成果物は共有カタログに入り無料ユーザーにも開放**＝「有料だけが作品を使える」形にならないことが立て付けの要。
  有料ユーザーのリクエスト＝最も信頼できる需要シグナルという副産物つき。
- 有料セットは4本に: 語彙リユニオン／AI解説（上限+BYOK）／統計深掘り（分析側）／**優先リクエスト**。
- ⚠公開直前の弁護士単発レビューで、価格ページ文言と合わせて確認項目に含める（content寄りの軽度グレーのため）。

### 4. 初期カタログ（marketing選定・実配信確認済み）
基準: 学習メディア言及頻度×JP配信可能性×SRT入手性×B1-B2帯×話数×感情強度。
- **必須トップ**: Suits（既存シード）/ Stranger Things / Breaking Bad / The Crown / Emily in Paris /
  Cobra Kai / プラダを着た悪魔（Disney+・**2026年5月続編公開の便乗**）/ Modern Family / The Good Place。
- **フレンズは初期除外**（Netflix日本から2022年配信終了→U-NEXT/Hulu対応フェーズまで保留。
  user-voiceの「Suits またはフレンズ」条件は Suits で充足）。
- 11位以下の軸: Virgin River / The Lincoln Lawyer / SHOGUN（S2便乗）/ ONE PIECE実写（S2便乗）等。
- オンボーディングでカタログ内作品への導線を一本引く（初日の成功体験＝リテンション最大因子）。

### 5. 段階構築（legal R2の線引き・契約前の許容ライン）
- **事前一括バッチ（数百話を数日で）は不可**＝OS商用契約後（フェーズ2）まで凍結。
- 契約前は**段階構築**: オーナーがアプリの通常経路で生成・**週30話上限**・
  **単一作品を短期集中で埋めない**（複数週に分散・「普通の視聴ペース」の外形維持）。
- 既存ガードレール継続: 内部解析限定・非配信・生SRT非返却・TTL/エビクション・表示1文＋出所明示。
- ⚠**層1（OS ToS）の既存注意**: VIP枠での共有キャッシュ配信は規模が上がるほど商用流用の外形が濃くなる
  （カタログ戦略が作った穴でなく既存の緊張・OSの救済は申告→削除型でソフト）。
  **限定公開の小規模のうちは黙認ライン・公開拡大の前にOS商用契約の交渉着手を前倒し**。
  最終判断は公開直前の弁護士単発レビューに含める。

### 6. 告知運用（marketing）
- **週1回・月曜更新**をリズム化（アプリ内バナー主軸・プッシュは許可者に週1まで・低露出方針中はXよりアプリ内優先）。
- リサーチ接続: コンテンツ鮮度がD30を支える（research-2026-07-sla-retention-growth §B-3）＝
  「継続の理由を毎週一個補充する仕組み」。
- **便乗カレンダー**: 続編・新シーズン公開に合わせて前作を対応（プラダ2/SHOGUN S2/ONE PIECE S2等）。
  外部イベントの話題性を予算ゼロで借りる。Google Trendsで事前予測・半期分の告知スケジュールを組める。
- 法務ガード: 非提携の近接明示・作品名は指示的言及のみ・ロゴ/公式素材/予告編不使用・
  「公式/公認/タイアップ」禁止・「◯◯の公開に合わせて配信」でなく「◯◯に関連する語彙教材をご用意」
  （作品本体の配信と誤認させない）。

### 7. 見せ方（品質保証への再フレーミング）
- 内部言語「コスト管理」を外部言語「学習効果が確認された作品を丁寧に整備」へ（実態が伴うので誇大でない）。
- カタログ数は前面に出さない。「英語学習者が実際に使っている人気作品に対応。毎週拡大中。リクエスト受付中」。
- 競合ポジション: LR（全対応だが品質崩壊報道あり）とLingopie（カタログ制・入口が厳しい）の中間＝
  「厳選＋無料で始められる＋リクエストで育てる」。
- ローンチメッセージ案: **「観たいドラマが、自分だけの単語帳になる。毎週、学習対応作品を拡大中。」**
- ⚠「毎週拡大中」は運用の確約が伴う（止まった時の信頼毀損）＝§3のSLAコミットと同根。

## 合理性・転換率の比較評価（2026-07-06 追記・オーナー問いへの回答）

### 合理性
数字はカタログ優位が明確（+¥8k→+¥14k・最悪シナリオ消滅・転換率が2割下振れしても元戦略の
現実シナリオと同等の頑健性）。本質は**「金銭リスクを運営コミットに交換する取引」**：
唯一の弱点は週次SLAという人的単一障害点（元戦略には無い）。

### サブスク加入率への影響（正直な評価）
- プラス: キャッシュ深化→体験濃度→D7改善（転換の先行指標）／週次鮮度→D30改善／優先リクエスト（補助）。
- 🔴**最大の盲点**: 転換の旗艦=語彙リユニオン**A案（近接予告）はカタログ依存**（今夜の話の語彙キャッシュとの
  交差で動く）。カタログ外の作品を観るユーザーには旗艦が発動せず、サブスクゲート（初回発動後提示）が
  永遠に来ない＝この層の転換確率がほぼゼロに落ちる。
- 総評: 転換率は「維持〜微増だが母数と旗艦の発動範囲が削れる」。**カタログ制の優位は転換でなく
  コスト側から来ている**。

### 改善5点（優先順・採用とセット）
1. 🔴**リユニオンB案（視聴直後リキャップ）を先行実装**（採用の実質条件）: B案は保存語＋履歴ベースで
   **全作品で動く**→旗艦とサブスクゲートの発動をカタログ非依存化。A案（近接予告）はカタログ内の
   上乗せ体験に格下げ（growth-features-strategy の「A案最初」の実装順をこの決定で反転）。
2. **SLAの自動化・期待値設計**: 個別の「翌週対応」約束でなく「**毎週月曜にまとめて追加**」の定期便形式。
   リクエスト自動返信＋対応予定一覧の自動生成で、運営作業を週1回・30分のバッチに固定。
3. **需要の先回り**: 限定公開ユーザーの視聴履歴から「リクエストされる前に対応」。
4. **D0救済のオンボ**: 視聴中作品がカタログ外の初日に「①クリック保存は全作品で今すぐ使える
   ②リクエスト1タップ ③カタログのおすすめ」の3点セットを必ず提示。
5. **公開拡大前の実測ゲート**: カタログ内/外ユーザーの D7・転換意欲を分けて計測し、
   カバレッジ投資（30→50→100作品）をデータで決める。
