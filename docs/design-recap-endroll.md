# 視聴直後リキャップの設計（語彙リユニオンB案の本格面・2026-07-13 /decide討論の結論）

user-voice × engineer の R1 独立見解が主要争点でほぼ一致収束（R2省略・小分岐のみ司会統合）。
前提: B案原則（実際に起きた再会を祝う・全作品で動く・アプリ内計算完結・拡張無改修）・
映画館コンセプト（薄い背骨＋節目だけ濃く）・罪悪感UI禁止・API追加ゼロで成立。

## 討論の合意（両部署一致）
- **自動の全画面ジャックは禁止**。「視聴直後」を確実に判定できない以上（savedAtは日付のみ）、
  誤爆（一気見の話間・翌朝に割込）が1回起きるだけで信頼を落とす。
- **主戦場はホームカードの深化**（全画面は任意到達＋本物の節目のみ）。
- **当日券は発行する。ただし格下げ版**（予習=特等席の差を死守）。
- **reunion-card と HomeStubCard の併存は統合**（ホームの clutter 解消）。
- **再会ゼロの夜は何も出さない**（「実際に起きたことだけ祝う」誠実さがブランド。
  無理な励まし演出=劣化Duolingo化）。

## 確定設計

### 1. 検知＝「再訪二段ゲート」（誤爆の構造的回避）
- savedAt が日付のみ＝視聴終了の瞬間は原理的に検出不可。代わりに
  **「保存 → 一度アプリを離れる → 戻ってくる」を視聴済みの強シグナル**とする。
- 実装: 未リキャップの保存グループを初検知した起動で `cl_recap_firstseen_{epKey}` を打刻するだけ
  （この起動では従来の再会カード表示のまま）→ **次回以降の起動で「観たあと」状態に昇格**。
  冪等ゲート `cl_recap_seen_{epKey}` で一度きり。
- epKey は半券と同キー体系に（保存語は tmdbId を持たないため `myDramas.find(title).tmdbId` で補完。
  title 割れによる二重発火を防止）。
- 却下案: 拡張からの視聴終了シグナル（B原則違反＋SW失効/タブ閉じの不確実性）・
  tsSec×作品尺比較（dual-path-trap の信頼度問題）。

### 2. 体験の形＝ホームの「観たあとに」統合カード
- 現在併存する reunion-card（語との再会）と HomeStubCard（半券・観たあとに）を
  **1枚の「観たあとに」カードへ統合**:
  - 予習済み作品 → 半券（聞きどころ）体裁 ＋ 再会チップ（『Breaking Bad』S2E1以来 等）
  - 予習なし作品 → 当日券体裁 ＋ 再会チップ
  - CTA: 「この◯語をさっと復習」＋（節目のみ）「エンドロールを見る」
- トーンは祝いのみ。観なかった日・翌朝でも文言は変えない（責めない）。

### 3. エンドロール（全画面・シネマ暖色）＝節目限定・任意到達
- **自動表示しない**。カードからのタップ到達＋「本物の節目」でのみ提案的に出す:
  シーズン完走／30日ぶりの再会／語数マイルストーン。スキップ常時1タップ。
- 実装: PrepLaunch に variant `'recap'` を追加し `.prep-cinema-*` 暖色資産（プレミアパス系）を流用。
- 内容: 今夜の語がロール→再会語がハイライト（場面つき）→締めは launch ramp 系の前向き文言。
  引用は1文厳守＋出所明示（48条）を演出内でも維持。
- 「毎回エンドロール」は映画館コンセプト（節目だけ濃く）の自己矛盾として明示的に却下。

### 4. 半券接続＝当日券（walk-in ticket）
- 予習なしでクリック保存だけした作品にも「観た証」を発行 → クリック保存だけのユーザーにも
  半券コレクションが育つ（カタログ外作品でも貯まる＝D0救済と整合）。
- **格下げの明確化**（報酬インフレ防止・予習の特等席を死守）:
  `kind:'walk-in'`・座席番号なし（「自由席」表記）・聞きどころ刷り込みなし・
  コレクション画面で紙質/縁色を変えて視覚区別。listens 不要。
- 発行タイミングは**明示アクション起点**（さっと復習 or エンドロール開始時）。
  カード表示の副作用で発行しない（openのたび発券される事故防止）。
- 同一エピソードに予習券がある場合は予習券優先（二重発行しない）。

### 5. 復習接続＝「さっと復習」（司会統合の分岐解消）
- 対象: エピソードグループ全語（再会語＋新語）・**上限5語**。
  再会語は「見出しの感動」でありフィルタではない。
- 形式: 既存 ReviewModal をそのまま使う（採点1タップが保存直後語の SRS 初期化を兼ねる＝
  getDueReviewWords が無srs語をdue扱いする実装が根拠・特別処理不要）。
  上限5語なら数十秒で終わり「さっと」の看板を守れる。
- グループが5語超のときは末尾に「もっと復習する」で本式復習へ逃がす。

### 6. 無料/Pro線
- **無料死守**: 統合「観たあとに」カード＋再会チップ＋さっと復習＋当日券（点火役）。
  「再会が起きたこと自体」を隠す人質型Proは印象最悪＝禁止（既定の倫理線どおり）。
- **Pro（予約席）**: 旗艦リユニオン専用画面（作品横断の再会履歴/統計）・エンドロール演出強化・
  検出範囲拡大（withinDays 延長・件数緩和）。件数は見せて中身をぼかすまで。

## 出荷計画（engineer見積・API追加ゼロ）
- **MVP（0.5〜1日）**: computeRecap 拡張（group/epKey 返却）＋再訪二段ゲート＋
  カード統合の骨格＋さっと復習の全語化（上限5）
- **P2（1〜2日）**: 当日券発行（issueWalkInTicket）＋2カード統合の完成＋コレクション画面の区別表示
- **P3（2〜3日）**: エンドロール（PrepLaunch variant 'recap'）＋節目判定＋isPro 接続（課金開始時）

- エンドロールの是非はペルソナ間で価値観が割れやすい論点（user-voice 注記）→
  実ユーザーが増えたら「エンドロールを見る」CTA のタップ率で実測判断
