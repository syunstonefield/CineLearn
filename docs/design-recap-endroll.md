# 視聴直後リキャップの設計（語彙リユニオンB案の本格面）

- 初版 2026-07-13: /decide討論（user-voice×engineer R1一致収束・R2省略）で「再訪二段ゲート」方式で確定
- **改訂 2026-07-19: 検知を「視聴完了の自己申告（質問カード）」へ差し替え（オーナー発案）**。
  きっかけはオーナー指摘2つ — ①エンドロールの入口がカードのみ＝見逃すと二度と見られない消失ガチャ
  ②「視聴完了を本人に教えてもらえば、検出問題ごと消える。Duolingoの1日のレッスンのような儀式にできる」。
  R1討論の合意（自動全画面ジャック禁止・カード主軸・当日券は格下げ版・全語復習・再会ゼロ非表示・無料死守）は
  すべて維持。変わったのは検知方式と、それに伴う発券タイミング・エンドロール入口のみ。

前提: B案原則（実際に起きた再会を祝う・全作品で動く・アプリ内計算完結・拡張無改修）・
映画館コンセプト（薄い背骨＋節目だけ濃く）・罪悪感UI禁止・API追加ゼロで成立。

## 確定設計

### 1. 検知＝視聴完了の自己申告（質問カード）
- 未申告の保存グループがある状態でアプリを開くと、ホーム最上部に質問カード:
  **「『作品名』SxEx、観終わりましたか？」＋保存語数チップ**。
- ボタンは2つだけ: **「観終わった」（主・金アクセント）／「まだ途中」（従・静か）**。
  「まだ途中」はそのセッションでは引っ込み、次のセッションでまた静かに聞く。
- **無視してよい**: 催促・赤バッジ・「未記録N件」表示は禁止（罪悪感UI禁止の憲法）。
  答えるか、次の視聴グループが来たら黙って置き換わる。
- **申告タップ＝視聴記録（epKey＋日時）として保存**。話数単位の視聴完了データが初めて手に入る
  （将来のリユニオン精度・統計・推薦の材料）。
- 誤爆は構造的にゼロ（祝いは必ず本人の申告の後にしか来ない）。
- 保存直後の同一セッションでは質問を出さない（視聴中にアプリを開いたケースへの配慮。閾値は実装時調整）。
- epKey は半券と同キー体系（保存語は tmdbId を持たないため `myDramas` から補完。title割れ防止）。
- 却下・格下げ案:
  - **再訪二段ゲート（初版採用案）**: 「保存→離れる→戻る」を視聴済みシグナルにする受け身検知。
    全員に自動で届く利点はあるが、祝いの前に確認が挟めず消失ガチャの根も残る。
    **質問カードの応答率が公開後の実測で低かった場合の保険として温存**（無視されたら翌セッションで
    受け身の再会カードに格下げ表示する追加案も同枠）。
  - 拡張からの視聴終了シグナル（B原則違反＋SW失効/タブ閉じの不確実性）
  - tsSec×作品尺比較（dual-path-trap の信頼度問題）

### 2. 体験＝「観たあとに」カードの2状態
- 現在併存する reunion-card（語との再会）と HomeStubCard（半券・観たあとに）を**1枚に統合**し、
  2状態を持たせる:
  - **質問状態**: 上記の視聴確認。演出なし・静か。
  - **祝い状態**（「観終わった」タップ直後に同カードが変化）:
    チケット獲得演出（prepTicketEarn 流用・タップから300ms以内）→ 再会チップ
    （『Breaking Bad』S2E3以来 等）→ CTA「この◯語をさっと復習」＋（節目のみ）「エンドロールを見る」。
- **祝い状態v2（2026-07-20・オーナーのモック発案→/decide R1で減量採用）: 二段構え**。
  タップ直後のみリッチ版＝全語の単語カード（再会だけ金バッジ＋出どころ行・NEWは無印・
  発音ボタン=lib/speak.js・4語以上で横カルーセル）＋可変統計「新規/再会/覚えてきた」
  （覚えてきた=isLearned(2回以上正解)・0の項目は非表示。「マスターに近づいた」は
  視聴申告がSRSを進めない以上不誠実なので不採用）。**再訪時は軽量版（再会チップ＋CTA）に畳む**
  ＝毎晩ミニエンドロール化してP3節目演出の希少性を殺さない。
  「視聴履歴を見る」リンクは行き先画面が無いため見送り→P2で当日券が入り
  「半券コレクション＝視聴の記録」が成立した時に半券タブ行きとして導入。
- 一気見: 未申告グループが複数話あるときは「S2 E1〜E3、観終わりましたか？」とまとめて聞き、
  1タップで一括申告・一括発券（P2）。
- 再会も保存も無かった夜は何も出さない（実際に起きたことだけ祝う）。
- トーンは祝いのみ。観なかった日・翌朝でも文言は変えない（責めない）。

### 3. エンドロール（全画面・シネマ暖色）＝節目限定・入口は二本
- 発火は**本物の節目のみ**: シーズン完走／30日ぶりの再会／語数マイルストーン。自動全画面は禁止・
  スキップ常時1タップ。
- **入口①: 申告直後の祝い状態からの提案**（「エンドロールを見る」）。
- **入口②: 半券コレクションからの再上映**（一度発火した節目は半券に「もう一度見る」が残る＝
  上映記録として恒久到達可能。消失ガチャの解消・P3）。
- 実装: PrepLaunch に variant `'recap'` を追加し `.prep-cinema-*` 暖色資産を流用。
- 内容: 今夜の語がロール→再会語がハイライト（場面つき）→前向きな締め。引用は1文＋出所明示（48条）を
  演出内でも維持。「毎回エンドロール」は節目の希少性を殺すため明示的に却下。

### 4. 半券接続＝当日券（walk-in ticket）
- **発行タイミング＝「観終わった」申告タップに統一**（明示アクション起点の完成形）。
  カード表示の副作用で発券しない。チケット＝**本人が確認した視聴の記録**という一貫した意味づけになる。
- 格下げの明確化（報酬インフレ防止・予習の特等席を死守）:
  `kind:'walk-in'`・座席番号なし（「自由席」表記）・聞きどころ刷り込みなし・
  コレクション画面で紙質/縁色を変えて視覚区別。listens 不要。
- 同一エピソードに予習券がある場合は当日券を出さない（予習券優先・二重発行なし）。

### 5. 復習接続＝「さっと復習」（変更なし）
- 対象: エピソードグループ全語（再会語＋新語）・上限5語。
- 形式: 既存 ReviewModal 流用。採点1タップが保存直後語の SRS 初期化を兼ねる
  （getDueReviewWords が無srs語をdue扱いする実装が根拠・特別処理不要）。
- グループが5語超のときは末尾に「もっと復習する」で本式復習へ。

### 6. 無料/Pro線（変更なし）
- 無料死守: 質問カード＋祝い状態＋再会チップ＋さっと復習＋当日券。
- Pro（予約席）: 旗艦リユニオン専用画面（作品横断の再会履歴/統計・エンドロール再上映棚もここに）・
  エンドロール演出強化・検出範囲拡大。件数は見せて中身をぼかすまで。

## 出荷計画（改訂・API追加ゼロ）
- **MVP（0.5〜1日）**: 質問カード（単一グループ）＋申告→祝い状態＋視聴記録の保存＋
  さっと復習の全語化（上限5）＋カード統合の骨格
- **P2（1〜2日）**: 当日券発券＋コレクション画面の区別表示＋一気見の一括申告
- **P3（2〜3日）**: エンドロール（節目判定＋半券棚からの再上映）＋isPro 接続（課金開始時）
